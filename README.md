# share-map-app

share-map-app は、複数の地点と道路沿いルートを地図上でまとめて指定し、共有URLとして渡せる静的Webアプリです。

この README は、先に利用者向けの外部仕様を説明し、その後に開発者向けの構成と作業ルールを説明します。

## 利用者向け仕様

### できること

このアプリでは、次の情報を地図上にまとめて表示できます。

| 種別 | 内容 |
| --- | --- |
| 地点登録 | 地図クリック、場所検索、緯度・経度入力で地点を登録 |
| ピン種類 | 出発、到着、立ち寄り、駐車場、注意、食事などを選択 |
| コメント表示 | 各ピンに地図上コメントを常時表示 |
| ルート表示 | 登録地点の順番に沿って道路ルートを表示 |
| 順番変更 | 地点一覧からルートの立ち寄り順を上下に変更 |
| 共有URL | 地点データを含むURLをコピーして共有 |

アプリはブラウザだけで動作します。利用者登録やログインは不要です。

### 画面構成

| 領域 | 内容 |
| --- | --- |
| 地図 | OpenStreetMap ベースの地図、ピン、コメント、ルートを表示 |
| 検索 | 地名や施設名で場所を検索 |
| 選択中の地点 | ピン選択時に地点名、種類、コメントを表示 |
| 地点編集 | 場所名、ピン種類、コメント、緯度、経度を入力 |
| ルート表示 | 道路沿いルートの表示切替と状態表示 |
| 共有リンク | 現在の地点一覧を含む共有URLをコピー |
| 地点一覧 | 登録済み地点の確認、選択、編集、順番変更、削除 |

右上の閉じるボタンでメニューを非表示にできます。メニューを閉じた後もピンは保持され、ピンをクリックするとメニューが再表示されます。

### 地点の追加

地点を追加する主な方法は次の通りです。

| 方法 | 操作 |
| --- | --- |
| 地図クリック | 地図上の任意の場所をクリックして座標を入力 |
| 場所検索 | 検索欄に地名や施設名を入れて検索結果の座標を入力 |
| 直接入力 | 緯度・経度を手入力 |

新規追加モードで座標が入ると、保存前の位置確認用に薄い仮ピンが表示されます。仮ピンはドラッグでき、動かすと緯度・経度欄も更新されます。

「地点を追加」を押すと正式なピンとして登録され、共有URLにも反映されます。

### 地点の表示と編集

ピンまたは地点一覧の「表示」を選ぶと、地点の表示モードになります。

表示モードでは、選択中の地点情報を確認できます。内容を変更したい場合は「編集」を押して編集モードへ移ります。

編集後に保存しても、編集対象の地点は保持されます。新しい地点を追加したい場合は「新規追加に戻る」を押します。

### ピン種類

| 種類 | 用途例 |
| --- | --- |
| 出発 | 集合場所、出発地点 |
| 到着 | 目的地、最終地点 |
| 立ち寄り | 途中で寄る場所 |
| 駐車場 | 車を停める場所 |
| 注意 | 注意点、危険箇所、補足地点 |
| 食事 | レストラン、休憩場所 |

ピン種類は色と記号で区別されます。

### コメント表示

地点ごとに「地図上コメント」を入力できます。

コメントを入力すると、地図上のピン付近にラベルとして常時表示されます。集合時間、入口の場所、注意事項など、共有相手が地図を見た瞬間に分かる情報を書く用途を想定しています。

### ルート表示

「道路沿いのルートを表示」をオンにすると、2地点以上のときに登録順で道路ルートを表示します。

地点一覧の上下ボタンで順番を変えると、ルートもその順番で再計算されます。

ルート計算は公開 OSRM API を利用します。外部APIの混雑、通信エラー、対象地域、経路条件によってはルートを表示できない場合があります。

### 共有URL

「コピー」を押すと、現在の地点一覧を含む共有URLをコピーします。

地点データはURLの `#places=` 以降に含まれます。共有されたURLを開くと、同じ地点、ピン種類、コメント、順番が復元されます。

### ローカル保存

このアプリは、現在のブラウザに地点一覧を一時保存します。

| 保存先 | 内容 |
| --- | --- |
| URLハッシュ | 共有URLに含める地点データ |
| localStorage | 同じブラウザで再表示したときの下書き地点データ |

別の端末や別のブラウザに共有する場合は、共有URLを使います。

### 運用上の注意

- 共有URLを知っている人は地点情報を閲覧できます。
- URL内に地点名、コメント、緯度、経度が含まれるため、個人情報や非公開情報を書かないでください。
- 地図タイルは OpenStreetMap、地図表示は Leaflet、ルート計算は OSRM の公開サービスに依存します。
- 外部サービスの制限や障害により、地図やルートが表示できない場合があります。
- 静的Webアプリのため、サーバー側に地点データは保存しません。

### 主な処理の流れ

#### 初期表示

1. `#places=` がURLにある場合は、URL内の地点データを読み込む
2. URLに地点データがない場合は、localStorage の下書きを読み込む
3. 地点一覧、ピン、コメント、ルート、共有リンクを描画する
4. 地点がある場合は、地図表示範囲を登録地点に合わせる

#### 地点登録

1. 地図クリック、場所検索、または手入力で緯度・経度を入力する
2. 新規追加モードでは、薄い仮ピンを表示する
3. 「地点を追加」で `state.places` に地点を追加する
4. localStorage と URL ハッシュを更新する
5. ピン、地点一覧、共有URL、ルートを再描画する

#### 地点編集

1. ピンまたは地点一覧から地点を選択する
2. 表示モードで地点内容を確認する
3. 「編集」でフォームに既存値を入れる
4. 保存すると同じ地点を更新する
5. 編集後も選択中の地点を保持する

#### ルート表示

1. 「道路沿いのルートを表示」がオンか確認する
2. 有効な地点が2件以上あるか確認する
3. 登録順の座標を OSRM API に渡す
4. 返却された座標を Leaflet の線として描画する
5. 順番変更、地点削除、地点編集時に再計算する

### データ形式

内部の地点データは次の項目を持ちます。

| 項目 | 内容 |
| --- | --- |
| `title` | 地点名 |
| `note` | 地図上コメント |
| `type` | ピン種類 |
| `lat` | 緯度 |
| `lng` | 経度 |

共有URLでは、URLを短くするために短いキーへ変換してから Base64 形式で `#places=` に入れます。

| 共有用キー | 元の項目 |
| --- | --- |
| `t` | `title` |
| `n` | `note` |
| `y` | `type` |
| `a` | `lat` |
| `g` | `lng` |

### テスト観点

変更内容に応じて、次の観点を確認します。

| 領域 | 確認内容 |
| --- | --- |
| 地点追加 | 地図クリック、検索、手入力で座標が入り、仮ピンが表示されること |
| 仮ピン | 新規追加前だけ薄いピンが表示され、保存後に消えること |
| 地点編集 | 表示モードから編集でき、保存後も選択地点を保持すること |
| ピン種類 | 種類ごとの色と記号が反映されること |
| コメント | 入力したコメントが地図上ラベルに表示されること |
| ルート | 2地点以上で登録順の道路ルートが表示されること |
| 順番変更 | 上下ボタンで地点順が変わり、ルートも更新されること |
| 共有URL | コピーしたURLで地点、種類、コメント、順番が復元されること |
| メニュー | 閉じるボタンでメニューが消え、ピン選択で再表示されること |
| レスポンシブ | デスクトップとモバイル幅で地図と操作パネルが破綻しないこと |

### 変更手順

1. GitHub の `main` ブランチを正式な原本として扱う
2. 作業場所は `C:\Users\kazts\CodexProjects` 配下に置く
3. 変更はまずローカルファイルだけに行う
4. 変更後にローカル表示と diff を確認する
5. 変更内容と確認結果を利用者へ提示する
6. 利用者が明示的に承認するまで、GitHub Pages へ反映しない

承認前に行ってはいけない操作は次の通りです。

- `git push`
- GitHub Pages へ反映される `main` ブランチへの push
- GitHub Actions の手動デプロイ実行

### GitHub Pages への反映ルール

原則として、反映順序は次の通りです。

1. ローカル変更を確認する
2. diff と確認結果を提示して承認を得る
3. GitHub へ commit / push する
4. GitHub Pages の公開URLで新しいファイルが配信されていることを確認する
5. 必要に応じて公開URLで主要操作を確認する

静的Webアプリのため、GAS やサーバーへのデプロイ作業はありません。

### 秘密情報の扱い

このアプリはAPIキーやサーバー側の秘密情報を使いません。

次の情報は Git に保存しません。

- 個人の住所や集合場所など、公開したくない地点情報
- 個人情報を含む共有URL
- GitHub 認証情報
- ブラウザやローカル環境の認証ファイル
